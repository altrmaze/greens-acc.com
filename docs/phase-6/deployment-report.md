## Security Posture

- All admin routes gated by `AdminRoute` → requires Supabase session + `admin` role from `profiles` table (server-side RLS)
- Role never derived from browser storage or JWT claims alone
- `DeveloperRoute` gates developer-only routes
- RLS enabled on all Supabase tables (migrations 04, 05, 07)
- No `ALLOW ALL` policies
- `VITE_APP_PASS`, `VITE_TEST_USER_EMAIL`, `VITE_TEST_USER_PASSWORD` injected only during build; not shipped as plain text
- `ErrorBoundary` wraps all admin section output — prevents runtime crashes from leaking sensitive stack traces
